# 测试说明

本文档说明当前项目的自动化测试范围、运行方式和已覆盖的核心风险点。

## 运行命令

```bash
npm test
npm run lint
npm run build
```

`npm test` 使用 Node.js 内置测试运行器，并通过 `tsx` 直接执行 TypeScript 测试文件：

```bash
node --import tsx --test --test-concurrency=1 tests/*.test.ts
```

测试没有新增第三方测试框架依赖。

## 测试文件

| 文件 | 类型 | 覆盖重点 |
|---|---|---|
| `tests/scoring.test.ts` | 单元测试 | 信用风险评分引擎、等级阈值、时间衰减、一票否决、维度扣分上限、许可资质与信用修复事件 |
| `tests/auth.test.ts` | 单元测试 | 签名会话令牌、篡改令牌拒绝、过期令牌拒绝、角色权限判断 |
| `tests/data-rating.test.ts` | 集成测试 | Prisma 数据访问、仪表盘统计、企业列表筛选、详情解析、模糊查询、重新评级、预警联动、级联删除 |
| `tests/test-db.ts` | 测试辅助 | 创建临时 SQLite 测试库，隔离测试数据，避免污染本地 `dev.db` |

## 已覆盖业务场景

### 评分评级

- 在营企业无风险事件时为 A 级 100 分。
- 行政处罚、抽查检查、投诉举报按事件严重程度扣分。
- 事件发生时间越久远，扣分按时间衰减。
- 85、70、60 分边界分别映射到 A、B、C 等级。
- 低于 60 分映射到 D 等级。
- 注销、吊销企业会在基础经营维度扣分。
- 单个维度扣分不会扣成负数。
- 一票否决事件会直接把企业评为 D 级，并将分数限制在 40 分以内。
- 许可资质事件作为信息性事件，不影响评分。
- 信用修复事件不会让信用记录维度超过满分。

### 权限和会话

- 可以生成并验证签名会话令牌。
- 篡改后的令牌会被拒绝。
- 缺失、格式错误和过期令牌会被拒绝。
- `admin` 和 `inspector` 拥有写权限，`viewer` 只读。
- 只有 `admin` 拥有管理员权限。

### 数据访问和联动

- 首页统计能基于企业最新评级生成等级分布。
- 待处置预警数量和风险事件总数能正确统计。
- 企业名录能返回最新评级、风险事件数和预警数。
- 企业名录支持按等级、行业和关键词筛选。
- 企业详情能解析评级明细 JSON。
- 模糊查询支持企业名称和统一社会信用代码。
- 重新评级会追加 `RatingRecord`，保留评级历史。
- C 级企业自动生成中风险待处置预警。
- D 级或一票否决企业自动生成高风险待处置预警。
- 重复重新评级不会重复生成同等级待处置预警。
- 删除企业时，风险事件、评级记录和预警记录会级联删除。

## 测试数据隔离

集成测试使用 `tests/test-db.ts` 创建临时 SQLite 文件，并通过 `DATABASE_URL` 指向该临时数据库。测试结束后会断开 Prisma 连接并删除临时数据库文件。

因此自动化测试不会修改项目根目录下的 `dev.db`，也不会影响答辩演示数据。

## 当前未覆盖范围

- 前端组件交互测试暂未引入 React Testing Library 或 Playwright。
- DeepSeek 相关 AI 接口依赖外部 API Key 和网络环境，当前不纳入默认自动化测试。
- Server Actions 的端到端权限流程目前通过权限纯函数、数据层集成测试和生产构建间接验证。

这些范围后续可以根据上线要求继续补充端到端测试。
